# Contributing

Contributions are welcome. The project is small, hand-authored, and tightly
coupled to upstream documentation; please read this guide before opening a
pull request.

## Ground rules

- **The YAML spec is the source of truth.** Never edit `references/*.md`
  directly — they are generated by `rtr generate` from `assets/spec/`.
- **CamelCase is non-negotiable.** Field names on the wire are always
  camelCase. Anything else is a spec bug.
- **Fidelity over completeness.** If a field cannot be confirmed against the
  live HTML, leave it out and open an issue rather than guessing.
- **Conservative edits.** Match surrounding style. Avoid introducing new
  dependencies without discussion.

## Development setup

```bash
git clone https://github.com/makafeli/realtime-register.git
cd realtime-register
npm install
npm run build
node bin/rtr.js --help
```

Requirements: **Node.js 20.11+**.

## Making a change to the spec

1. Identify the operation in `assets/spec/<category>.yaml`.
2. Run `node bin/rtr.js scrape <operationId>` to pull the current live HTML.
3. Diff the printed skeleton against the YAML entry.
4. Edit the YAML. Match the structure documented in
   [`docs/spec-format.md`](docs/spec-format.md).
5. Run the full verification:
   ```bash
   npm run build
   node scripts/audit-refs.mjs
   node bin/rtr.js doctor
   node bin/rtr.js generate
   ```
6. Commit the YAML change **and** the regenerated `references/*.md` in the
   same commit.

## Making a change to the CLI

1. Edit the relevant file under `src/cli/commands/`.
2. Keep `bin/rtr.js` untouched — it is a shim that resolves `dist/` or `src/`.
3. `npm run build` to produce fresh `dist/` output.
4. Add a CHANGELOG entry under `[Unreleased]`.

## Commit style

- Imperative mood: "Add X", "Fix Y", "Reconcile Z".
- Reference operation IDs where relevant:
  `brands.yaml: reconcile createBrand required fields`.
- Group related changes into one commit; keep mechanical renames separate
  from semantic edits.

## Pull request checklist

Before opening a PR, confirm:

- [ ] `npm run build` succeeds with no TypeScript errors.
- [ ] `node scripts/audit-refs.mjs` reports `problems: 0`.
- [ ] `node bin/rtr.js doctor` reports all 200s.
- [ ] `references/*.md` is regenerated if any YAML changed.
- [ ] `CHANGELOG.md` is updated under `[Unreleased]`.
- [ ] No `console.log` or debug code in `src/`.

## Reporting drift

If upstream RTR docs change and a spec entry needs reconciling but you do not
want to open a PR, file an issue with label `fidelity-drift` and include:

- The `operationId`.
- The change you observed (new field, renamed path param, removed enum, …).
- A link to the live doc page.
